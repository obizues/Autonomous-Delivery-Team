# Screenshots Guide

Use this guide to capture and place project screenshots for public viewers.

## Folder

Store images in:

- `docs/assets/screenshots/`

## Recommended Set

1. `01-overview.png`
   - Dashboard Overview tab with workflow summary cards.
2. `02-process-graph.png`
   - Workflow Graph showing revision loops and gate decisions.
3. `03-engineer-control-tower.png`
   - Engineer control section with agents, lanes, and cross-review matrix.
4. `04-final-outcome.png`
   - Final Outcome section with approval and acceptance checklist.
5. `05-event-log.png`
   - Event log with transitions and decision events.

## Capture Tips

- Use consistent browser zoom (100%).
- Keep the same window width/height for all screenshots.
- Prefer clean demo runs with representative data.
- Avoid including secrets, tokens, or private repo URLs.

## Markdown Example

![Overview](assets/screenshots/01-overview.png)
